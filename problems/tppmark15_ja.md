第11回TPP (Theorem Proving amd Provers) 研究集会  
TPPmark11

命題論理式の標準形を定義し、それがいわゆる正規形と呼ぶに十分な要求を満たしていることを証明するのが今回の問題である。

## 問題

1. 命題論理式の全体が作る集合（あるいは型）を定義せよ。
2. 二つの命題論理式が同値である、という関係を定義せよ。
3. 命題論理式を受け取って命題論理式を返す関数 `nf` を定義せよ。ただし `nf` は次を満たすものとする。
   - 3.1 任意の命題論理式 `P` について `P` と `nf(P)` は同値である。
   - 3.2 `P` と `Q` が同値であれば、`nf(P)` と `nf(Q)` は構文的に全く等しい命題論理式である。
4. 3.1 と 3.2 を証明せよ。

## 出題の意図

こんな出題のしかたでは証明の対象が曖昧ではないか、と感じたかもしれない。しかし、今回は証明そのものだけでなく、論理式や真偽値をどう定義するか、どんな関数をどう定義するかといった問題の定式化（オントロジーの設定）も比較し合いたいと考えている。命題論理式の標準形に関する非形式的な議論にはある程度共通理解があっても、具体的にどのように定式化し形式化するかには様々な方法がある。使用する定理証明系やそのライブラリが定式化のしかたに影響を与えるかもしれない。形式的証明だけでなく、その定式化のしかたも含めて比べてみよう。

例えば以下の点で、定式化や証明の design decision が必要になると考えられる。

- 命題変数を無数に用意するのか、ある自然数 `n` について `n` 個の変数を用意し、議論全体を `n` でパラメータ化するのか。
- 同値関係や `nf` の計算を論理式の書き換えだけで構文的に定義するのか、意味論的対象（たとえば NBE）を導入するのか。
- `nf` を積和標準形にするのか、和積標準形にするのか、その他の形にするのか。
