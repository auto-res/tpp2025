# TPPmark17

今回は最長共通部分列（LCS: Longest Common Subsequence）に関する課題です。解答は指定された問い合わせ先まで提出してください。

## 定義

部分列は連続している必要はない。長さ `n` の列は最大で `2^n` 通りの部分列を持つ。例えば `ABC` の部分列は `ε`（空列）、`A`, `B`, `C`, `AB`, `BC`, `AC`, `ABC` の 8 通り。

有限列 `s` と `t` の共通部分列とは、`s` の部分列でもあり `t` の部分列でもある列。最長共通部分列とは、その中で最も長いもの。ただし一意とは限らない。例えば `AB` と `BA` は `ABA` と `BAB` に対する最長共通部分列としてどちらも成立する。

## 問題

1. 有限列 `s` と `t` を受け取り、最長共通部分列（の一つ）を返す関数 `LCS` を定義せよ。列は使用する証明支援系に応じたデータ構造（例: リスト）で表現してよい。要素の型も自然数など適宜固定して構わない。
2. `LCS s t` が `s` と `t` の共通部分列であることを証明せよ。必要なら「部分列であること」を表す述語を定義すること。
3. `LCS s t` が最長であること、すなわち `u` が `s` と `t` の共通部分列ならば `|u| <= |LCS s t|` を証明せよ。

### 補足

LCS を求める典型手法は動的計画法だが、ここでは実装方針を問わない。全探索でもよいし、証明が行いやすい方法を重視してもよい。
